## 5. Construction-collapse hypothesis — confirmed at TWO horizons

The hypothesis (per cycle #01 closeout §5 + Step 0 memo): long-only × monthly × top-N × 78-symbol universe collapses any signal into "rank by defensive winner-ness" because the construction is over-determined. Cycle #02 was the cleanest test of the joint half: shorter horizon (5d) + weekly cadence should let intraday / reversal / volatility factors compete.

**Result**: even at 5d horizon + weekly cadence, the top of the IC ranking is dominated by the SAME {β + mom_12_1 + volume} template. The construction-collapse is robust to horizon AND cadence axes — it's not a horizon artifact.

This narrows the next-cycle search axes:
1. **Long-only constraint** — the foundational invariant; can't be relaxed.
2. **Top-N selection mechanism** — could be replaced by sharpe-weighted, volatility-targeted, equal-risk, etc.
3. **Universe size (78 symbols)** — small/mid-cap could change construction dynamics, but data not available without polygon/yfinance subscription extension.
4. **Cross-asset / cross-sectional construction** — the composite is per-symbol; could become long/short within sectors (still long-only at portfolio level via cash buffer).

The first three axes are not viable in the next iteration without infrastructure work. The fourth (sector-relative construction) is the cheapest option and worth a Track C-3 cycle.

## 6. Critical addendum — harness realized-NAV invalid on production panel

The Step 1 harness (`core/research/harness/composite_evaluator.py`) passed 26 unit tests on synthetic data. When invoked on production prices via `BarStore.load(adjusted=True)`, NAV explodes to 10^100 magnitudes — Step 3a investigation traced this to **heterogeneous split adjustment in `data/daily/<sym>.parquet`**:

```
LRCX 2015-04-15..04-30 (RAW from data/daily/LRCX.parquet):
  04-15: $72.43       04-22: $6.72        ← 10x scaled
  04-16: $71.80       04-23: $77.07       ← back to non-scaled
  04-17: $70.66       04-24: $6.57        ← 10x scaled again
  04-20: $6.17 ←10x   04-28: $76.39       ← non-scaled
  04-21: $6.67 ←10x   04-29: $6.55        ← 10x scaled
                      04-30: $75.57       ← non-scaled
```

The parquet contains bars from multiple sources (likely polygon-flat pre-2024 vs trades-backfill or a different feed) merged without normalization. `splits.parquet` only handles forward splits — it can't repair this kind of source-mixing artifact (LRCX has only one 1:10 split entry, dated 2024-10-03).

Step 0 (RCMv1 retro β verification) hit the same issue and worked around it by reading paper-engine `live_like_pnl.csv` directly. For cycle #02 closeout this means:
- IC-based metrics (mining objective, family census, composite correlation) are valid
- Realized-NAV metrics (cum_ret, vs_spy, vs_qqq, MaxDD, stress-slice) are NOT computable via harness on production data

Cycle #02 still closes (0-nominee, Tier-2 factor-level sibling) because IC-level analysis is sufficient.

**Open follow-up** (task #49): audit all 78 symbols for this pattern; determine root cause; quarantine or repair affected parquets; restore harness production capability.
